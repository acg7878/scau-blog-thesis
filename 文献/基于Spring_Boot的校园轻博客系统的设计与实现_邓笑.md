Abstract
With the popularity of the Internet, the Internet has become the most convenient and
fastest channel for people to collect information and communicate. The network also gives
people a stage to fully express themselves. In the context of such a nationwide exchange,
it is important to open up a network of exchanges between teachers and students
throughout the school, and to establish a simple and clean network platform for
communication between teachers, students, and between teachers and students.
meaningful. For this purpose, while the light blog has a variety of expressions and simple
social features, Spring Boot is a lightweight framework, with convenient development and
maintenance features, so designed and implemented a Spring Boot-based campus light
blogsystem.
According to the software life cycle process, the system first analyzes the
requirements of the system, analyzes the functions, performance and security requirements
of the system, then designs the system, and divides the whole system into system
management module, blog management module and label management module. ,
interactive management module and user information management module, and detailed
design of specific modules, and then use the development tools to achieve the system's
blog browsing, blog post search, blog post editing, as well as user management and blog
management and other core functions, and finally by means of software testing methods
Testthefunctionality,performance, and security ofyour system.
In general, the functions of the campus light blog system are basically realized, and
the performance and security are also guaranteed accordingly: the system uses the Spring
Boot+MyBatis development framework to make the code structure low-coupling, the
system is easy to develop and maintain; the front-end framework and many The plug-in
enables the system to support multiple publishing formats, with a simple interface and
simple operation; Apache Shiro security framework and filters and other technologies to
ensuresystem security.

1.1 研究背景与意义
高校中的学生正处于寻找自我的阶段，刚成年或已成年，思想或许正处于萌芽
期，急于向世界宣告自己独立而特别的存在；或许少年老成，已学会表达自我多时。
同时，如今的网络时代也已向人们提供了大量的方式去认识世界、感知世界，如微
博，刷一刷即可了解此刻最新最劲爆的消息；微信，摇一摇即可结识身边或近或远
的人们；抖音，拍一拍即可记录或感动或搞笑的瞬间。轻博客作为一种介于微博和
博客之间的一种网络社交模式，它既继承了博客的表达基因，注重内容为主以及兴
趣驱动，同时保留了微博的传播基因，保留用户之间的互动，包括评论、喜欢、转
发等特点，使其具有线上“俱乐部”之称，以兴趣主导，以精致表达吸引围观，进而
形成互相分享、互相学习、互相探讨的良好社交氛围。
高校的学生们所能获取信息以及表达自我的网络媒体通道有贴吧、BBS、微博
公众号、微信公众号、人人网、QQ群及其公众号等方式，但无一种是师生们能全体
参与并且自由表达的方式。贴吧多为学生“地下组织”，虽也有管理员管理，但信息
鱼龙混杂，有效信息很难获取；BBS虽含有大量信息，但因其编辑复杂以及其界面
较为陈旧，很难吸引学生长久驻留；各类公众号多用于发布通知，且具有一定的工
作性质，这使得公众号维护者必须定期更新，同时务必保证信息准确。
校园轻博客系统的意义主要体现在以下几个方面：
（1）轻博客帮助学生养成文字表达的行为习惯。轻博客作为比博客相对轻松，
比微博相对严肃的方式，用精致内容表达着对世界的认知，表达着对自我的探寻，
表达着对梦想的追寻，表达着对现实的抵抗。文字图片的排版以及内容编排需要静
下来去思考、去沉淀，这其实给了高校的学生们一种去浮躁的表达方式，也是帮助
学生们养成文字表达的行为习惯。
（2）校园环境使得交流更加安全可靠。通过表达认识一批志同道合的伙伴们，
这些小伙伴可能是比你小一两届的学弟学妹们，可能是比你高几届的学长学姐们，

1.2 国内外研究概况
博客，又称部落格，是从Blogger音译而来，Jorn Barger在1997年首次提出weblog，
Evan Williams在2006年创立了Twitter，David Karp在2007年创建了介于Twitter和
blogger之间的Tumblr[1-3]。在国内，2000年weblog的概念首次进入中国，2002年博客
因博客中国的开通从而进入全面推广时期；2005年各大门户网站纷纷构建自己的博
客系统，进入博客时代[4,5]。2007年开始出现像饭否等独立微博的踪影，2009年微博
因新浪微博的上线开始在全国流行[6]。2011年出现第一个轻博客——点点网，同一个
时期各大网站纷纷推出了自己的轻博客，如网易的乐乎，新浪Qing等，轻博客开始
受到关注[7,8]。
博客在教育应用中占据其独特优势。首先，博客打破了话语权的绝对权威性，
在传统的教学模式下，老师具有绝对的权威性，学生的表达多半会被忽略，甚至可
能出现嘲笑，严重时可能出现被孤立等情况产生，而博客因其表达性和匿名性，可
以有效的进行讨论相关争议，并能吸引“支持者”，同时引起教育者的关注，并获得
平等的对话权；其次，博客能够促进Learner to Learner的学习模式，每个人都可以是
老师，每个人都可以参与其中，共同学习，共同进步；再次，博客给予了学生尝试
网络学习的指导性，帮助学生在纷繁复杂的网络世界中找到有效信息；最后博客促
进了学生的成长，记录下自己的所思所感，并借助记录可以进行一个不断反思自省
的阶段，进而获得成长[9,10]。
博客对校园文化也产生了一定积极影响。校园文化，作为一种特殊的社会文化，
它将校园作为活动范围，校内人员作为活动主体，围绕校园精神开展校园活动。校
园文化作为学校的重要组成部分，由学校师生共同创造，并且体现着学校对外展现

博客在校园中的发展已有一段时间，足以说明博客与校园的结合是可以被接受，

但这样的博客大多数是以老师或专业人士为主，以学生为辅，师生的互动性以
及学生参与其中的主动性相对而言并没有达到预期的要求。轻博客作为博客和微博
的延申，虽同样强调内容，但更多以兴趣驱动，鼓励用户主动的去了解，去学习，
去交流所喜欢的、所感兴趣的内容，只有当学生主动参与其中，其互动性才会得到
实质性的进展。同时，作为新一代的社交模式，轻博客在很多方面都表现出了自己
的独特性：
（1）重图片轻文字，阅读体验更轻松：图片表达所带来的视觉冲击性以及生动
形象性，使得浏览图片比阅读文字更适合当代人文化消费的快节奏，阅读体验更轻
松；
（2）重内容轻关系，致力于高质量内容的发布与传播：以兴趣为驱动的社交，
相对更强调内容而非人本身的设计模式使得内容的发布更个性化，内心驱动使得内
容更高质量化、更自主创作化；
（3）信息需求趋同，形成深度交流和共建关系：通过轻博客发现志趣相投的人
们互动会更加频繁，关系相对更加紧密[15]。
轻博客与校园的结合弥补了校园博客环节中所缺失的学生自主表达的环节，并且
在某种程度上拉近了高校中学生与学生之间的距离。

3.4.1 数据库建模说明
根据本章3.1-3.3 节的需求分析说明，系统将涉及到三个主要实体，分别是用户、
博文以及标签；多个与互动相关的实体，分别是评论、喜欢、推荐，以及关注、通
知、黑名单。
实体之间的关系分为四种：一对一、一对多、多对一以及多对多。
1）首先说明三个主要实体之间的关系：
（1）用户与博文的关系为一对多：一个用户可以拥有多篇博文，而一篇博文只
会有一个用户（作者）；
（2）博文与标签的关系为多对多：一篇博文可以拥有多个标签，而一个标签也
可以标记多篇博文；
2）然后说明在用户实体之间的互动操作关系：
（1）通知：一个用户可以收到多条通知；而一个用户（管理员）可以发出多条
通知；
（2）黑名单：一个用户可以将多个用户加入黑名单，一个用户也可以被多个用
户加入黑名单；
（3）关注：一个用户可以关注多个用户，一个用户也可以被多个用户关注；
3）接着说明在用户实体和博文实体之间的互动操作关系：
（1）评论：一个用户可以评论多篇博客，一篇博客也可以被多个用户评论；
（2）喜欢：一个用户可以喜欢多篇博客，一篇博客也可以被多个用户喜欢；
（3）推荐：一个用户可以推荐多篇博客，一篇博客也可以被多个用户推荐；
4）最后说明回复评论与评论以及用户之间的关系：
（1）评论：一个用户可以对多个评论进行回复，一个评论也可以被多个用户回
复；
（2）回复评论：一个用户可以对多个回复进行再一次的回复，一个回复也可以
被多个用户进行再一次的回复。

4.2.1 系统公共部分说明
系统的文件结构大致共包含 5 个包：com.blog.notices、com.blog.blogs、
com.blog.labels、com.blog.interactions、com.blog.systems，每个包下主要包含 controller、
service、dao、domain 这四大类。controller包存放 XXController类，负责前端与后台
的交互；service 包存放XXService 接口和它对应的 XXServiceImpl 实现类，负责业务
逻辑；dao 包存放XXDao 接口，负责与数据库交互；domain 包存放对象类，如User。
除这四大类以外，有的包中可能还有一些其他的类，如util 包——用于放置工具类、
config 包——用于放置配置类、aspect 包——用于放置切面类、exception 包——用于
放置异常类等等。
系统采用thymeleaf 模板渲染作为view 部分，采用thymeleaf 作为模板语言，用
来接收返回的模板数据。view 到controller 的连接需要在 pom 文件中引入相关依赖，
在配置文件中添加相关配置，在模板中加入标签库，编辑模板视图。
系统采用大量注解，为 controller 类采用@Controller 注解，为 service 类采用
@Service注解，为dao类采用@Mapper注解。在controller层还采用@RequestMapping
等相关注解定义请求路径和参数等数据，采用@RequestBody 或@ResponseBody 将数
据以固定格式（json格式）的获取或返回，采用@RequiresPermissions 等限制访问接
口的用户权限。在 service 层还可以采用@Transactional 进行事务管理。在 dao 层也
可采用@Select、@Insert、@Update 或@Delete 等注解直接在对应接口上定义 sql 语
句，但本系统采用mapper.xml的形式定义数据库操作。
系统采用mybatis 数据持久层框架和 MySQL 数据库，采用SQL 语句的形式定义
数据库中数据的CRUD操作。配置方面需在 pom中引入相关依赖，需进行数据源配
置以及mybatis 的基本配置，统一采用 dao 对应的 mapper.xml 来编写 SQL 语句，并
进行相关mapperScanner和mapping 的配置来实现dao 和mapper.xml对应关系。
对于日志和缓存等一类具有“横向”特性的功能，则利用 spring 的AOP（面向切
面编程的特性）的注解@Aspect、@Pointcut、@Before 等定义相关日志操作。

4.3.2 xss内容过滤
为防止常见的 xss 跨站脚本攻击，系统设置了 xss 过滤器对用户提交数据，如
博文或评论，进行检查和过滤，并借助 Jsoup 工具的 HTML 文档清理功能来做进一
步的过滤工作。
在使用 Filter 之前需要进行相关配置，系统采用注解，非 xml 配置的方式创建
了 XssConfig 类作为配置类，主要需要配置 filter 的注册类 FilterRegistration- Bean，
设置 filter 的配置参数，如 filter 实例，初始化参数，filter 拦截的 url 地址等等。
系统通过定义 XssFilter 以及 XssHttpServletRequestWrapper，以及借助 Jsoup 工
具定义一些过滤接口，存放在 JsoupUtil 类中，这里主要介绍 XssFilter 类和
XssHttpServletRequestWrapper类的内容。
（1）XssFilter类
XssFilter继承了Servlet的Filter类，可以对request和responce进行过滤，XssFilter
重写了doFilter()，init()，destroy()方法，init()方法主要初始化一些配置中的预置参数，
比如预置过滤内容，destory()方法是在 filter 工作结束后处理的工作，doFilter()方法
则是核心的过滤的操作，所以这里主要说明doFilter()方法。

系统测试
4.4
系统测试是指基于一些规定条件来操作程序以检测程序的错误，测量软件质量
并评估其是否满足设计要求的过程。系统测试的目的是为了找到系统运行过程中不
符合系统需求的地方。
系统测试主要通过大量的测试用例对系统的功能点进行逐个检测，大致通过三
个方面去检测系统是否达到预期的要求：检测该实现的功能是否及时执行，该提示
的错误信息是否正常显示，该应对的安全防御是否正确执行。
测试方法
4.4.1
系统主要基于白盒测试和黑盒测试进行测试。白盒测试，又称透明盒测试，结
构测试等，是基于程序内部结构的测试，主要通过检查代码在条件，路径，溢出等
句法上的错误来指出结构上的缺陷。黑盒测试，又称功能测试，数据驱动测试等，
是基于系统外在功能的测试，主要通过有效输入和无效输入来验证系统功能，以及
外部结构是否正确反馈。

本文围绕主题——基于Spring Boot 的校园轻博客系统的设计与实现，进行了三

终：从整体来说，论文的大体结构则是系统的核心内容是什么，为什么要设计这样

为什么要做需求分析，以及需求分析应该怎么做；从某一个点来说，以Apache Shiro
技术为例：Apache Shiro 是什么，为什么要用 Apache Shiro，怎么通过 Apache Shiro
来达到最终目的？
本文在绪论章节中通过校园轻博客系统的研究背景与意义，国内外研究情况等

问题；而系统需求分析和设计以及系统实现和测试章节重点描述了校园轻博客系统
的“怎么做”过程，也是系统开发过程的几大重要步骤：
