基于 Spring Boot 的多用户博客系统的设计研究

摘 要

伴随着信息时代的发展和即时通讯工具的普及，传统媒体和网络平台也经过

了不断的改革和创新。如今博客应用成为了继邮件、BBS 论坛、即时通讯软件之

后的又一大互联网交流媒介，这顺应了当下互联网积极发展生态的趋势：即通过

结合多样化的元素生成各种服务于不同行业的特色博客平台。虽然我国的互联网

起步晚于国外，技术上也略微弱势于国外，但是这并不妨碍我国网民的热情。我

国早已成为世界上网民最多的国家，这反映了我国积极对外的社交政策，同时也

达到了宣扬民族文化自信的目的。

系统在 B/S 架构上基于 Spring Boot 的后端框架进行开发；通过 FreeMarker

模板引擎实现表示层与业务逻辑层的架构分离；程序模块化设计，并明确前后端

开发人员具体业务划分。其中博文模块内置 Markdown 语法编辑器，提高用户输

入效率；针对用户登录，基于 OAuth 协议实现微博、QQ 三方登录功能;结合百度

OCR 文字识别实现文章多方式输入以及系统检索功能；最后使用 Lucene 实现对

全文的模糊查询。同时体系上使用 MVC 分层设计，使得系统更加安全科学，并且

实现域名访问博客系统。

本文首先通过相应需求分析对系统进行了总体的设计，将整个多用户博客系

统划分为六大子模块：分别为博文管理模块、个人主页管理模块、标签管理模块、

互动管理模块、用户信息管理模块、系统管理等模块，并对具体的模块进行详细

的设计，实现了系统的博文发布、博文浏览、博文搜索、博文编辑，图片上传和

博文删除以及用户管理和博客管理等核心功能，最后借助动态测试方法对系统进

行测试。本文探讨的博客不论在内容属性上还是技术开发中都是多样化的：从早

期记录的网络日志到如今 IT 技术知识的分享、从好友圈里的心情记录到后来社

区论坛的话题参与等。在生活中不管是技术人员还是普通用户，都喜欢在博客圈

子里发帖或者评论；一方面可以记录用户当时的一些观点想法，吸引更多志同道

合的人参与讨论；另一方面，事后每次阅读还会产生新想法，在知识话题的逐渐

积累后，通过再次整理博文，就会具有更多的收获。

写博客是一种督促人进步的好习惯，也是个人思想认知得以传播的过程。不

仅在内容上记录个人生活的精彩瞬间或者是点点滴滴，更加体现在通过内容帮助

别人的一种价值体现。所以，本文研究与设计的博客系统具有较高使用价值和长

远意义。

关键词：Spring boot 框架，Markdown 解析，Lucene 搜索，知识分享

Abstract

messaging tools, traditional media and network platforms have undergone continuous

reform  and  innovation.  Today,  blog  application  has  become  another  Internet

communication medium after email, BBS forum and instant messaging software, which

conforms to the current trend of the positive development of the Internet Ecology: that

is, through the combination of diverse elements to generate a variety of featured blog

platforms for different industries. Although China's Internet started later than abroad,

and its technology is a little weak, But this does not hinder the enthusiasm of Chinese

netizens. China has long become the country with the largest number of Internet users

in the world, which reflects China's active external social policy and achieves the goal

of promoting national cultural self-confidence.

The system is developed based on the back-end framework of spring boot in B / S

architecture; the architecture separation of presentation layer and business logic layer

is realized through FreeMarker template engine; the program is designed in modularity,

and the specific business division of front-end and back-end developers is made clear.

Among  them,  the  blog  module  built-in  markdown  grammar  editor  to  improve  the

efficiency of user input; for user login, based on OAuth protocol to achieve the three-

way  login  function  of  microblog  and  QQ;  combined  with  Baidu  OCR  character

recognition to achieve multi-mode input of articles and system retrieval function; finally,

Lucene to achieve the fuzzy query of the full text. At the same time, the system uses

MVC layered design, which makes the system more secure and scientific, and realizes

the domain name access blog system.

First of all, this paper designs the whole system through the corresponding demand

analysis,  and  divides  the  whole  multi-user  blog  system  into  six  sub  modules:  blog

management  module,  personal  homepage  management  module,  tag  management

module,  interactive  management  module,  user  information  management  module,

system management and other modules, and designs the specific modules in detail to

realize the system's blog Publish, browse, search, edit, upload and delete images, user

management and  blog  management are the core functions of the system. Finally, the

system is tested with the help of dynamic test method. The blog discussed in this paper

is diversified in content attribute and technology development: from the early recorded

1.3 研究目的和意义

博客能够清晰表达出自身观点，且这个过程是开放自由的。但是在现阶段，

我们发现在很多博客论坛中，当你精准的寻找你想了解的知识时，往往会碰到难

题，现阶段很多网站的内容给人呈现的都是一种碎片化的观感，这让用户精准定

位内容的行为成本大大增加，同样也降低用户互联网冲浪体验感受，所以传统网

站过去只能按标题内容搜索的模糊查询已经渐渐满足不了当下这个知识爆炸的

社会。对于一个博客系统来说，用户想要查询站内博文，站内全文搜索功能是必

不可少的要求[18]。但是往往很多网站内容检索功能使用上并不如意，目前主流的

内容检索模块功能主要有两种：

（1）通过网站构建所运用数据库自带的索引功能。数据库管理系统中，常

用的 MySQL、Oracle 等,都是可以使用数据库自带的索引功能实现系统的全文搜

索，即通过相应索引机制达到筛选数据的功能。

（2）网络上使用相应的搜索引擎组件达到全文搜索。这种开发模式相对简

单，而且后期维护方便且支持结合开发人员自定义逻辑算法达到后期系统的趋于

完善[19-21]。本系统使用的 Lucene 搜索引擎就是这种典型的工具。

我们在深入探讨全文检索技术的时候，不得不了解博客知识分享的精髓理念。

现阶段的技术博客是对于专业知识管理研究最深入的一种类型，它不仅仅使得博

客文章之间加深了逻辑联系，同时其也在知识分享和社交方面趋于平衡，并且去

除大量冗余信息，要求内容上通俗易懂。在本文的博客系统中，为了让大家更好

2.1 Spring Boot 框架

Spring  Boot 框架是基于 Spring 框架发展而来，最初目的旨在简化原来

Spring 的复杂搭建问题，以及后面一系列的开发部署和调试。Spring Boot 的出

现，简化开发人员的操作，自动处理三方框架且基本不需配置 XML 文件[28]。通过

Maven 组件就可以解决依赖文件版本冲突的情况，大大提高开发人员的效率。

Spring Boot 支持热部署、配置检单、兼容多种数据库管理软件。选择这样一款

框架可以高效率的搭建出独立运行访问的程序项目。

2.1.1 Spring Boot 优点

（1）部署项目独立运转。Spring  Boot 可以将开发完备的程序直接打包成

jar 包快速上传服务器，方式和平时借助开发工具运行 java 项目没有任何不同，

降低程序员学习成本。

（2）本身内部内置了相关 WEB 服务器容器。Spring  Boot 内部提供 tomcat

等 servlet 访问容器，使得项目部署变得流程化，方便简洁。

（3）提供框架本身配置的对象模型（POMS）简化 Maven 配置，尊崇“约定

大于配置的理念”，使得开发过程尽量避免编辑配置文档。

（4）Spring Boot 不仅自动配置 Spring 容器，也可以根据框架提供的特性,

对于系统健康进行自检和生成外部配置工具。

2.1.2 Spring Boot 应用特性

Spring Boot 通过注入依赖来控制其反转特性，实现对象生命周期的容器管

(1) 登录和注册功能

用户通过注册，产生进入博客系统的唯一标识 ID，此后用户所有交互的功

3.5.2 安全性需求分析

户，采取为用户 ID 设置 cookie 唯一标识符的方法，用户在前台页面所有访问服

4.6 系统安全设计

本文开始对系统安全性进行了需求分析，本节对于系统的安全性方面进行了

如下设计：

（1）数据安全性。为解决开发设备因故障或者由于技术人员失误等造成数

28

基于 Spring Boot 的多用户博客系统的设计研究

据损坏，可以将数据存储在专用数据库的服务器上。

（2）系统采用分层设计结构，保证三层结构中表示层不能直接操作数据访

问层，即无法直接通过修改用户提交表单来操作数据库数据。

（3）系统为用户密码加密，将采用 2 次 MD5 散列算法对密码进行盐值加密，

此过程不可逆，从而保障了密码的安全传输。

（4）操作安全性。系统的操作部分，相应角色的用户才能对系统中所特有

的功能模块进行操作，保证了操作的规范性。

（5）系统设置数据的多层验证，通过 Ajax 来实现表单提交的基本数据格式

和简单逻辑验证。再结合 Shiro 安全框架，使用 Shiro 中的概念身份认证(登录)

和授权(权限验证)，通过 Shiro 中内置的过滤器，如 Authc:认证过滤器,其保证

只有登录之后才能访问；Perms:授权过滤器,其具有某种权限才能访问；Roles:

角色过滤器,其保证具有某种角色权限才能访问。最后在自定义的 Realm 里实现

授权，对系统用户角色进行相应的权限操作，符合相关业务逻辑[47]。

5.4 百度 OCR 文字识别的实现

博客系统在全文检索和文章编辑上传模块实现了基于百度 OCR 的文字识别

功能，使得输入功能可以多样化选择方式，便捷高效。

5.4.1 百度 OCR 文字识别实现原理

系统基于深度学习技术的百度文字识别实现这一功能。首先，通过上传图片

接口将图片传入后台服务器；接着获取百度文字识别应用的 access_token；然

后利用 access_token 来创建 HTTP 请求，将图片文件输入流通过 HTTP 请求访问

百度 OCR 的 general_basic；最后，解析请求成功后的 Json 数据，获取识别后

的结果。Json 数据中包含文字的具体信息，如位置，文字等。

5.4.2 百度 OCR 文字识别实现过程

首先登陆百度云，创建文字识别的应用，成功后我们可以得到 ID 以及 API

Key 和 Secret  Key，然后就可以调用百度 OCR 给自己开发的系统使用。具体如

下：在系统里面实现两个功能类，AuthService 类通过 API Key 和 Secret Key 获

取 Access_token，和服务器端获取连接，general 类调用系统相关接口实现功能，

6.1 全文总结

本文从当前互联网比较流行的环境生态和社交论坛开始，研究其在获取信息

方面不足的特点，通过结合博客技术上对于知识信息的解释与延展，生成全新的

博客类型，这是一个承载知识分享与社交功能的博客。主要叙述的是一个基于

Spring Boot 框架技术的多用户博客相关技术研究。系统使用了当前流行的前后

端分离技术，前端部分使用了自适应的 Bootstrap 框架,响应式的 css 能够自动

适配移动设备的屏幕界面,易于开发。后端采用 Spring Boot 轻量式开发框架,结

合 MVC 软件开发模式，模块分层处理业务逻辑，通过 Lucene 分布式搜索引擎对

博文进行索引构建然后达到博文检索功能，使得系统降低冗余，提高访问响应速

度。主要开展工作与研究总结如下：

（1）前台界面主要达到了一个展示布局与内容的效果。除了为游客和用户

提供主流相关知识分类、具体文字图片内容、在相应界面可对博文发表评论看法

外，同时博文作者进行必要的回复，鉴于此系统是多用户博客系统，可多人发表

博文，多人进行评论互动。

（2）后台主要是对前台用户操作相应的响应管理,除了注册用户、管理账户

信息、通过邮箱验证修改密码的功能、相应博文内容的整理编辑以及其标签的管

理设置，同时文章可在用户登录的基础上前后台都可进行撰写，其内置的

Markdown 语法编辑器简单方便，支持左右侧实时预览，不用担心前端博文出现

格式性错误。

（3）开发采用 Maven 项目管理工具，通过管理相应 jar 包部署程序测试，

生成站点日志，广泛运用在语法自检、服务启动、部署和发布。同时在程序实现

过程中，详细介绍 Spring Boot 框架在系统中的运用，Bootstrap 为前台提供接

口服务，探讨如何使用 MySQL 数据库为系统提供增删改查，并阐述其优势。

（4）在最后对于系统进行测试，我们通过相应测试方法对于系统进行系统

测试，方便快捷且行之有效的达到了对于各个模块功能的详细测试。同时针对于

系统实现过程中碰到的困难进行一一分析讨论，最后阐述相应解决方法。

综上，基于 Spring Boot 的多用户博客设计开发，通过把握内容与社交主题

的微妙平衡，致力实现为不同类型用户打造自我的生态圈。在目前整个社会追逐

现代简约风格与生活品质的当下，以博客系统为载体，给用户打造多样化的知识

汲取渠道，提高办公的效率，希望这样一个基于兴趣使然，用户可以静心沉淀自

己的博客社交圈能够得到大家的认可与青睐。

6.2 展望

基于 Spring  Boot 的多用户博客系统的实现采用了当下大热的 java 后端框

架，并且将其积极运用到设计开发中，博客系统一方面为用户提供撰写博文和生

态互动的渠道，同时也具备给予自己与别人相互分享所见所闻，感悟人生的意义。

虽然博客系统在使用具有可操作性，技术上也都具有一定研究价值，但是在项目

跟进的过程中，仍能发现一些不足和需要改进的地方。比如在后期博客使用上用

户和相应博文数量未能具有突破，面对内容丰富多样的数据来说，如何科学高效

的分析相应文章内容，对于相关语料有价值的部分可以进一步实现数据的可视化

分析，以下列举的方面也可以做的更好。

（1）全面的浏览器兼容性。因为这是一个网页端展示的 java web 项目，通

常在浏览器中生成界面，所以面对各种版本不一的浏览器必然会出现兼容性问题。

现阶段市面上主流的浏览器除了兼容性高的谷歌、火狐以及微软自带的 Edge 浏

览器，更存在许许多多内核驱动不一的浏览器，在这些小众浏览器上必然会出现

各种影响用户体验的一些 bug，带来各种视觉和人机交互上的差异体验。所以更

得差异化的检测这些浏览器兼容特性，追求完美。

（2）平台多端使用的博客社区。虽然采用 Bootstrap 框架具有在 PC、Pad、

手机端浏览器自适应调节界面的功能，但是毕竟设计只针对于浏览器实现，所以

现在考虑到手机的普遍应用，后期做一个适配于移动端的 app 和线上微信公众号

运营平台进行多端登录也是很有必要的。

（3）更加艺术唯美的前端显示效果。由于目前系统追求现代简约的风格，

重视模块内容和技术实现，所以在需求阶段对于前台界面的 UI 特性考虑的不是

很到位，没有实现更多的显示特效以及复杂功能，比如后期除了邮箱注册可以加

入手机验证码功能，改变其背景内容，完善页面交互性的视觉内容。

（4）平台运营完善机制。后期可以通过数据仓库分析，了解用户喜好比重，

实现内容上的个性化推荐功能，同时在现有百度 OCR 识图功能之上完善人机交互

的应用使得系统更加智能化。同时后期通过考量用户的定位，结合博客的社交属

性和相应元素标签，加工原有市场，探索新的盈利模式。

（5）系统安全保证。开发结束后部署服务器上传测试，选用成熟的阿里云

服务器，过程中因为相关高危漏洞打补丁的情况造成修复过程中数据丢失，相关

配置环境紊乱的问题，造成不得不格式化重新部署的问题，所以在之后信息应用

安全方面也要格外重视，可以加入更高的安全方案，使得系统稳健性大大提高。
